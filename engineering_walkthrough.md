# Phase 3: The Truth Layer (Evals Complete)

We have built the ultimate proof of quality for your Hybrid AI Router: an **Automated Evaluation Framework** with an **LLM-as-a-Judge**.

## 1. The Evaluation Dataset
I have created `evals/dataset.json` containing 15-20 diverse test cases, ranging from simple greetings to complex system architecture design. Each case has an "Expected Tier" assigned based on industry-standard complexity definitions.

## 2. LLM-as-a-Judge
Instead of you manually checking every answer, we use your **local Gemma 2 9B** to act as a "Judge."
- **Logic**: For every response generated by the router, the Judge is asked to grade it (1-5) on Accuracy, Helpfulness, and Depth.
- **Self-Correcting**: This creates a feedback loop where you can see exactly where the router is underperforming.

## 3. Automated Reporting
The `evals/eval_runner.py` script generates a professional Markdown report in the `evals/reports/` directory.

---

## How to Run the "Proof of Quality"

1. Open your terminal in the project root.
2. Run the evaluation script:
   ```bash
   python evals/eval_runner.py
   ```
3. **Wait for the results**: The script will cycle through each test case. You will see logs for "Testing Query..." and "Grading with AI Judge...".
4. **Read the Report**: Once finished, go to `evals/reports/` and open the latest `.md` file.

### What to look for in the report:
- **Routing Accuracy**: If this is below 80%, we need to adjust your Semantic Router thresholds in `router.py`.
- **Judge Score (Avg)**: If the average score is low (e.g., < 3.0), it usually means the RAG context wasn't relevant or the chosen model wasn't powerful enough.
- **Latency**: See exactly how much time your "Pro" routing adds compared to "Local."
